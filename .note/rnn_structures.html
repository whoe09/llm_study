RNN 구조 한눈에 정리

은닉 상태 · 다대일 · 일대다 · 다대다 — 비유와 흐름도로 이해하기

🧠 은닉 상태 (Hidden State)

RNN을 이해하는 출발점 — 모든 구조의 공통 기반

한 줄 정의: 은닉 상태 = RNN이 지금까지 읽은 내용을 요약해 들고 다니는 "기억 메모". 매 시점(단어)마다 이 메모를 읽고, 새 단어를 반영해 갱신한다.

📖 비유 1 — 책을 읽으며 메모하는 사람

소설을 한 문장씩 읽으며 작은 수첩에 줄거리를 요약한다고 상상해보세요.
  • 1번째 문장 → 수첩에 요약 작성 (= 은닉 상태 h₁)
  • 2번째 문장 → 기존 요약 + 새 문장을 합쳐 업데이트 (h₂)
  • … 마지막 문장까지 → 수첩엔 소설 전체의 요약이 담김
수첩 크기는 항상 똑같음 → 이게 바로 "은닉 상태의 크기 = 32". 문장이 아무리 길어도 32개의 숫자로 압축해서 들고 다닙니다.
이전 기억(hₜ₋₁) + 현재 단어(xₜ) → 새 기억(hₜ)

🏭 비유 2 — 컨베이어 벨트의 작업자

단어가 벨트를 타고 한 개씩 들어옵니다. 작업자(RNN 셀)는 ① 이전 작업자가 넘긴 메모(hₜ₋₁)를 받고 ② 새 단어(xₜ)를 보고 ③ 둘을 합쳐 새 메모(hₜ)를 만들어 다음 작업자에게 넘깁니다.
핵심 공식 (외울 필요 X): hₜ = tanh(hₜ₋₁ · Wₕ + xₜ · Wₓ + b)
→ "이전 기억"과 "현재 입력"을 섞어 "새 기억"을 만든다는 뜻.

📖 다대일 (Many-to-One)

여러 개 입력 → 결과는 하나 · 지금 학습 중인 감성 분석 모델

핵심: 단어를 여러 개 입력받아 문장 전체를 읽고, 결과는 하나만 출력 (예: 긍정 / 부정).
📝 감성 분석 (리뷰 → 긍정/부정) ⭐ 별점 예측 🏷️ 문서 주제 분류

📖 비유 — 책 다 읽고 한 줄 평

리뷰의 모든 단어를 순서대로 읽으며 수첩을 갱신하다가, 마지막 수첩 요약(h_last)만 꺼내 "이 리뷰는 긍정일까?"를 판단합니다. 중간 요약들은 최종 요약을 만들기 위한 과정일 뿐.
모든 단어를 읽은 뒤 마지막 은닉 상태만 시그모이드에 통과 → 긍정 확률 출력
이진 분류 & 시그모이드: 답이 두 개 중 하나(긍정 vs 부정)라 0~1 사이 확률 하나를 뱉는 시그모이드를 출력층에 사용. (이게 곧 로지스틱 회귀)

🖼️ 일대다 (One-to-Many)

입력 하나 → 출력 여러 개

핵심: 하나의 입력을 받아 → 여러 개의 출력(시퀀스)을 순서대로 생성.
🖼️ 이미지 캡셔닝 (사진 1장 → 설명 문장) 🎵 음악 생성 (시작음 → 멜로디)

🎤 비유 — 사진 한 장 보고 이야기 들려주기

그림 한 장을 슥 본 뒤, 그걸 머릿속에 담고 "강아지가… 잔디밭에서… 뛰고 있다" 처럼 단어를 한 개씩 이어서 문장을 만들어냅니다. 입력은 하나(사진)지만 출력은 여러 단어!
하나의 입력을 받아 단어를 한 개씩 이어 붙여 시퀀스를 생성

🗣️ 다대다 (Many-to-Many)

여러 입력 → 여러 출력 · 두 가지 유형으로 나뉨

🛂 유형 1 — 입력마다 즉시 출력 (동기형)

핵심: 단어를 하나 읽을 때마다 그 자리에서 바로 결과를 하나씩 출력. 입력 길이 = 출력 길이.
🏷️ 개체명 인식(NER) 🔤 품사 태깅(POS)
입국 심사대의 심사관 🛂 — 여행객(단어)이 한 줄로 지나가면, 심사관은 앞선 여행객 정보(이전 은닉 상태)를 기억한 채 지금 여행객을 보고 그 자리에서 바로 도장을 찍습니다(출력). 코드에선 return_sequences=True.
매 시점 입력(파랑) → 매 시점 출력(주황). 입력·출력 길이가 같음

🗣️ 유형 2 — 다 읽고 → 다 출력 (인코더-디코더 / seq2seq)

핵심: 입력을 끝까지 다 읽어 요약한 뒤, 그 요약으로 새 시퀀스를 처음부터 생성. 입력 길이 ≠ 출력 길이여도 OK.
🌐 기계 번역 💬 챗봇 📰 문서 요약
통역사 🗣️ —
  • 1단계 듣기(인코더): 한국어 문장을 끝까지 다 듣고(중간에 말 안 함) 머릿속에 "전체 의미"를 요약 = 문맥 벡터.
  • 2단계 말하기(디코더): 그 요약을 바탕으로 영어를 한 단어씩 생성. 방금 뱉은 단어를 보며 다음 단어를 이어감.
듣기 담당(인코더) + 말하기 담당(디코더), 두 개의 RNN으로 구성!
인코더가 다 읽어 문맥 벡터로 요약 → 디코더가 그걸 받아 새 문장을 한 단어씩 생성

📊 한눈에 비교

은닉 상태 관점 — "언제, 몇 번 꺼내 쓰느냐"의 차이

구조입력 : 출력비유출력 시점대표 예시
다대일다 : 1책 다 읽고 한 줄 평마지막 1번감성 분석
일대다1 : 다사진 보고 이야기매 시점이미지 캡셔닝
다대다 ①다 : 다 (길이 같음)입국 심사관매 시점 즉시개체명 인식
다대다 ②다 : 다 (길이 달라도)통역사다 읽은 후 생성번역 · 챗봇
은닉 상태로 정리:
  • 다대일 → 마지막 수첩 요약 하나만 꺼내 씀
  • 다대다 ① → 매 시점의 수첩 내용을 그때그때 출력으로 변환
  • 다대다 ② → 인코더의 마지막 수첩을 디코더에게 건네주고, 디코더가 새 수첩을 써 내려감
은닉 상태 개념 자체는 동일하고, 구조마다 사용 방식만 다를 뿐!
